# 비커넥트랩 - 청년 사용자 페이지별 기능 명세

## 페이지 맵

```
apps/web
├── / (리다이렉트 → /announcements)
├── /login                      # 로그인
├── /login/email                # 이메일 로그인
├── /register                   # 회원가입
├── /register/social            # 소셜 회원가입 (부가정보)
├── /reset-password             # 비밀번호 재설정 요청
├── /reset-password/confirm     # 비밀번호 재설정 완료
├── /announcements              # 공고 목록
├── /announcements/[id]         # 공고 상세
├── /apply/[announcementId]     # 지원서 작성
├── /apply/[announcementId]/complete  # 지원 완료
├── /mypage                     # 마이페이지
├── /mypage/edit                # 내 정보 수정
├── /about                      # 소개 페이지
└── /interviews                 # 인터뷰 목록
```

---

## 1. 로그인 / 회원가입

### 1.1 로그인 화면 (`/login`)

**구성 요소:**

- 카카오 로그인 버튼
- 구글 로그인 버튼
- 이메일 로그인 버튼

**동작:**

- 소셜 로그인 클릭 → OAuth 인증 → 기존 회원이면 로그인 완료, 신규면 `/register/social`로 이동
- 이메일 로그인 클릭 → `/login/email`로 이동

### 1.2 이메일 로그인 화면 (`/login/email`)

**입력 필드:**

- 이메일
- 비밀번호

**버튼:**

- 로그인 → 성공 시 메인 이동, 실패 시 "이메일과 비밀번호를 다시 확인해 주세요."
- 비밀번호 재설정 → `/reset-password`
- 회원가입 → `/register`

### 1.3 비밀번호 재설정 (`/reset-password`)

**Step 1 - 이메일 입력:**

- 이메일 입력 → 재설정 링크 발송 요청
- 발송 완료 안내 문구 + "이메일 다시 보내기" 버튼

**Step 2 - 이메일 내 링크 클릭 → `/reset-password/confirm?token=xxx`:**

- 새 비밀번호 입력 (8~16자, 영문+숫자+특수기호)
- 비밀번호 확인
- 변경 버튼 → 완료

### 1.4 회원가입 (`/register`)

**계정 정보 (필수):**

- 이메일 (형식 유효성, 중복 확인)
- 비밀번호 (8~16자 영문+숫자+특수기호)
- 비밀번호 확인

**부가 정보 (필수):**

- 이름 (실명, 텍스트)
- 휴대폰 번호 (숫자)
- 생년월일 (숫자)
- 성별 (남/여 선택)

**부가 정보 (선택):**

- 관심지역 (드롭다운, 복수 선택)
- 희망직무 (텍스트)
- 보유역량 (텍스트)

**이용약관 동의:**

- 필수: 만 14세 이상, 서비스 이용약관, 개인정보 수집 및 이용
- 선택: 마케팅 이용 동의
- 전체 동의 체크박스
- 각 약관 문서 링크

**완료 버튼:** 필수 항목 모두 입력 + 필수 동의 체크 시 활성화

### 1.5 소셜 회원가입 (`/register/social`)

- 이메일 정보는 소셜에서 제공 (수정 불가)
- 부가 정보 입력 + 이용약관 동의 → 이메일 회원가입과 동일

**API:**
| Method | Endpoint | 설명 |
|--------|----------|------|
| POST | `/auth/login` | 이메일 로그인 |
| GET | `/auth/kakao` | 카카오 OAuth |
| GET | `/auth/google` | 구글 OAuth |
| POST | `/auth/register` | 이메일 회원가입 |
| POST | `/auth/register/social` | 소셜 회원가입 (부가정보) |
| POST | `/auth/reset-password` | 재설정 링크 발송 |
| POST | `/auth/reset-password/confirm` | 비밀번호 변경 |
| POST | `/auth/refresh` | 토큰 갱신 |
| POST | `/auth/logout` | 로그아웃 |

## 2. 공고 목록 (`/announcements`)

**접근:** 로그인 불필요

**정책:**

- 게시 상태 = `게시`인 공고만 노출
- 모집 완료 공고도 노출하되 지원 불가

**목록 항목 표시 정보:**

- 썸네일 이미지
- 모집 상태 배지 (썸네일 위)
- D-day 배지 (모집 종료일 기준)
- 공고명
- 지역
- 혜택 정보 칩

**정렬:** 등록일시 기준 내림차순

**API:**
| Method | Endpoint | 설명 |
|--------|----------|------|
| GET | `/announcements` | 공고 목록 (게시 상태만) |

---

## 3. 공고 상세 (`/announcements/[id]`)

**접근:** 로그인 불필요

**표시 정보:**

- 공고 기본 정보: 공고명, D-day, 주최, 주관, 활동 지역, 활동 기간, 모집 기간, 혜택 정보
- 공고 상세 내용 (HTML)
- 이후 일정 정보: 결과발표, 사전교육, 현장체류 (입력된 항목만 표시)

**지원하기 버튼:**

- 기본 상태: 비활성화
- 활성화 조건: 게시=게시 AND 모집=모집중
- 클릭 시:
  - 로그인 상태 → `/apply/[id]` 이동
  - 비로그인 → `/login`으로 이동 (리턴 URL 포함)

**API:**
| Method | Endpoint | 설명 |
|--------|----------|------|
| GET | `/announcements/:id` | 공고 상세 |

## 4. 지원 화면

### 4.1 지원서 작성 (`/apply/[announcementId]`)

**접근:** 로그인 필수

**정책:**

- 동일 공고 중복 지원 가능
- 제출 후 수정 불가

**입력 필드:**
| 필드 | 필수 | 설명 |
|------|------|------|
| 이름 | O | 자동 입력, 수정 불가 |
| 이메일 | O | 자동 입력, 수정 불가 |
| 연락처 | O | 자동 입력, 수정 불가 |
| 지원서 파일 | O | 파일 업로드 |
| 기타 파일 | X | 파일 업로드 |
| 아웃바운더 인지 경로 | X | 단일 선택 |

**제출 버튼** → 지원 완료 페이지 이동

### 4.2 지원 완료 (`/apply/[announcementId]/complete`)

**표시:** 공고명 포함 안내 문구

**버튼:**

- "접수 확인하기" → `/mypage`
- "다른 프로그램 둘러보기" → `/announcements`

**API:**
| Method | Endpoint | 설명 |
|--------|----------|------|
| POST | `/applications` | 지원서 제출 |
| POST | `/files/upload` | 파일 업로드 |

## 5. 마이페이지

### 5.1 내 정보 확인 (`/mypage`)

**접근:** 로그인 필수

**a. 프로필 정보:** 프로필 이미지, 이름, 이메일, 연락처, 성별, 나이, 거주지, 활동 상태

**b. 직무 및 역량:** 관심 지역, 희망 직무, 보유 역량

**c. 계정 정보:** 가입 일시, 마케팅 이용 동의, 탈퇴 안내 문구, 로그아웃 버튼

**d. 공고 지원 이력 테이블:**
| 컬럼 | 설명 |
|------|------|
| 공고 명 | |
| 직무 | |
| 활동 지역 | |
| 지원 일자 | |
| 첨부파일 1 | 클릭 시 다운로드 |
| 첨부파일 2 | 클릭 시 다운로드 |
| 지원 상태 | |

**e. 참여 활동 테이블:**
| 컬럼 | 설명 |
|------|------|
| 프로그램 명 | |
| 활동 지역 | |
| 활동 기간 | |
| 참여 상태 | |

> 참여 활동은 지원 상태가 `최종 합격`이 된 시점에 생성

**f. 내 정보 수정 버튼** → `/mypage/edit`

### 5.2 내 정보 수정 (`/mypage/edit`)

**수정 가능 항목:**

- 프로필: 프로필 이미지, 이름, 연락처, 거주지, 활동 상태 (드롭다운)
- 직무/역량: 관심 지역 (드롭다운, 복수 선택), 희망 직무, 보유 역량
- 계정: 마케팅 이용 동의 (토글)

**버튼:**

- 취소 → 마이페이지로 이동
- 저장 → 확인 다이얼로그 "저장하시겠습니까?" → 확인 시 업데이트 + 토스트 "저장이 완료되었습니다."

**API:**
| Method | Endpoint | 설명 |
|--------|----------|------|
| GET | `/users/me` | 내 정보 조회 |
| PATCH | `/users/me` | 내 정보 수정 |
| GET | `/users/me/applications` | 지원 이력 |
| GET | `/users/me/activities` | 참여 활동 |

**API:**

|--------|----------|------|
| GET | `/interviews` | 인터뷰 목록 |
